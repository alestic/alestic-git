Alestic Git Server - Start Instructions

You have started an EC2 instance running a Git server with gitolite using an AMI built by Alestic.com (Eric Hammond).

You can get started managing your Git repositories and access control by cloning the gitolite admin repository. Your command will look something like this: 

    git clone git@YOURHOST:gitolite-admin

Update the command to use the current IP address of your EC2 instance.

The recommended way to run an Alestic Git server is to first upload your personal ssh key to Amazon EC2 and then specify that key when you run the Alestic Git AMI. If you started an Alestic Git AMI with a different EC2 keypair, then you will need to update your $HOME/.ssh/config file to include lines like the following. Make sure you specify the correct location of the EC2 ssh key file. 

    Host YOURHOST
        IdentityFile EC2KEYPAIR.pem

You will find it easier to use you Git+gitolite server if you associate an Elastic IP address with it and add a hostname in your DNS.
