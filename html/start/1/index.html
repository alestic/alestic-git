Alestic Git Server - Start

First Steps

You have started an EC2 instance of the Alestic Git Server using an AMI built by Alestic.com (Eric Hammond).

You can start managing your Git repositories and access control by cloning the gitolite admin repository from your server. Your command will look something like this: 

git clone git@YOURHOST:gitolite-admin

Update the command to use the current IP address of your EC2 instance.

Once you have a copy of the gitolite admin repository, you can continue following the gitolite documentation for adding users and repos.

The gitolite software relies on ssh to control access to your Git repositories. If your git clone command above gave you an ssh error, please read the next section carefully.

ssh Access

The recommended way to run an Alestic Git server is to first upload your personal ssh key to Amazon EC2 and specify that key when you run the Alestic Git AMI.

If you did not follow those instructions and you want to run an Alestic Git AMI with a different EC2 keypair, then you will need to update your $HOME/.ssh/config file to include lines like the following: 

Host YOURHOST
    IdentityFile /home/.../EC2KEYPAIR.pem

Make sure you specify the correct location of the EC2 ssh key file and the correct IP address or hostname for your instance.

Hostname

You will find it easier to use you Alestic Git server if you associate an Elastic IP address with the EC2 instance and add a hostname for the IP address in your DNS (e.g., git.example.com).
